Features

Everything you need to orchestrate agents

Agent Ensemble

LLM, ReAct, Sequential, Parallel, and Loop agents. Chain a researcher into a writer, run 3 analysts in parallel, or loop a coder with a reviewer until tests pass.

Event Streaming

Async event-driven architecture with real-time token streaming. Every agent yields typed events — partial tokens, tool calls, tool responses, and final answers — observable from any parent.

Composer

Declare entire agent orchestras in YAML — no Python wiring needed. Define tools, models, planners, and multi-agent pipelines in a single file. Run with orx my-agents.yaml.

Tools

20+ built-in tools: filesystem with offset/limit, shell exec, background tasks, sleep, tool search, memory, and agent-as-tool. Each tool caps its own output to prevent context overflow.

Auto-Memory

Persistent per-project memories that survive across sessions. 4 typed categories (user, feedback, project, reference) with YAML frontmatter, auto-maintained index, and verification nudges.

Planners

TaskPlanner injects task status into every LLM prompt and keeps the agent working until all tasks complete. PlanReActPlanner adds structured chain-of-thought reasoning.

MCP & A2A

First-class Model Context Protocol for tool servers and Agent-to-Agent protocol for cross-service agent communication. Expose any agent as an A2A server with one flag.

29 LLM Providers

OpenAI, Azure OpenAI, Anthropic, Google, Mistral, Cohere, Groq, DeepSeek, Ollama, and 20 more. Auto-detects provider from model name. Switch mid-session with /model.

How it works

Three steps to orchestration

Define

Create agents with tools, instructions, and an LLM. Each agent is a Python class or YAML block.

Compose

Wire agents together — sequential pipelines, parallel fan-out, review loops, or transfer routing.

Run

Interactive CLI, A2A server, or programmatic async streaming. One command: orx

Build agents in minutes

Create powerful AI agents with just a few lines of Python. Stream events in real-time and compose complex orchestration patterns.

  • 29 LLM providers with auto-detection
  • Persistent auto-memory across sessions
  • Dark/light themes, thinking spinner, tool approval
  • Session management and memory built-in
  • Type-safe with full Pydantic models
agent.py 
from orxhestra import LlmAgent, Runner
from langchain_openai import ChatOpenAI

agent = LlmAgent(
    name="assistant",
    model=ChatOpenAI(model="gpt-5.4"),
    instructions="You are a helpful assistant.",
)

runner = Runner(agent=agent, app_name="my-app")

async for event in runner.astream(
    user_id="user-1",
    session_id="session-1",
    new_message="Hello!",
):
    print(event.content)

Declare. Compose. Run.

orx.yaml 
# Define an entire coding agent in YAML
defaults:
  model:
    provider: openai
    name: gpt-5.4

tools:
  filesystem:
    builtin: "filesystem"
  shell:
    builtin: "shell"

agents:
  planner:
    type: llm
    instructions: Output actionable steps for the coder.

  coder:
    type: llm
    tools: [filesystem, shell]
    instructions: Execute the plan. Never ask the user.

  dev_loop:
    type: loop
    agents: [coder, reviewer]

  coordinator:
    type: sequential
    agents: [planner, dev_loop]

main_agent: coordinator
